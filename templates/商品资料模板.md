# 商品资料

> 测试资料包用途：用于 Product MCP L2 真实商品创建闭环验收，并作为最新 `CommoditySaveDTO` 创建商品字段范本。文件路径均为相对本 Markdown 文件所在目录的相对路径。

## 0. 填写规则

1. 不需要手动查系统 ID。分类、单位、供应商、区域、客户类型等优先填写名称，Codex 会通过 MCP 只读工具查真实 ID。
2. 文件类字段填写相对路径，不填写 OSS URL。Codex 会先核实文件存在，再调用 MCP 上传到 OSS，最后在提交 payload 中用 OSS URL 替换相对路径。
3. 路径相对于本 Markdown 文件所在目录。推荐写法：`./图片/主图.png`、`./认证/CE证书.pdf`。
4. 多个文件一行一个，或用表格多行填写。不要把多个路径写在同一个单元格里。
5. 金额、重量、尺寸、数量、天数只写数字，不带单位。
6. 是否类字段只填：`是`、`否` 或留空。Codex 会转换为 `1` / `0`。
7. 日期统一填写 `YYYY-MM-DD`。
8. 状态字段支持：`上架`、`下架`、`作废`。默认使用 `上架`；只有用户明确要求作废状态时才填写 `作废`。
9. 不确定的字段可以留空。不要编造 HS 编码、价格、认证编号、供应商、租户、创建人等业务数据。
10. 创建真实商品前，Codex 必须先输出提交摘要，由用户确认后再上传文件和调用创建接口。

### 0.1 多选字段填写规则

多选字段统一使用表格多行或列表一行一个选项。

示例：

```text
经销客户
工程项目客户
OEM/品牌客户
```

### 0.2 多文件字段填写规则

多文件字段必须一行一个文件，或表格一行一个文件。

示例：

| 图片用途 | 文件路径 | 标题 | 描述 |
|---|---|---|---|
| 细节图 | ./图片/细节图1.png | 正面细节 |  |
| 细节图 | ./图片/细节图2.png | 背面细节 |  |

### 0.3 文件路径与上传后替换规则

Codex 创建商品前必须完成：

```text
相对路径 -> 本地绝对路径 -> product_upload_file -> OSS URL -> product_create payload
```

### 0.4 必填项清单

必填清单为 DTO（`CommoditySaveDTO`）与前端保存前硬拦截要求的并集。通用硬阻塞项：商品中文名称、产品类型、上架状态、一级分类、计量单位、供应商、适用范围、商品主图，以及销售交付售后中的是否类开关。

条件硬阻塞项：

- 产品类型为整机时：产品等级、参考成本价 人民币、利润率 %、至少一行基础配置必填；已填写的基础配置/技术参数必须补全配置值/参数值。
- 产品类型为整机或配件时：包装长 mm、包装宽 mm、包装高 mm、包装费、包装重量 kg、净重 kg 必填。
- 适用范围为指定区域时：区域明细至少一行，且必须可解析区域。

商品英文名称与 Banner 图仍为可选。产品型号可选，但填写时仅允许英文大小写、数字、空格，且不能有首尾空格、中文或标点符号。
