# 商品资料

> 本文档用于维护 ERP 商品创建资料。请只填写真实、可确认的商品信息；不确定的字段保持空白。文件类字段填写相对路径，不填写 OSS URL、网络 URL、本机绝对路径或 base64。

## 0. 填写规则

1. 本模板按 ERP 新增/编辑商品页签组织：基础信息、产品配置、价格信息、库存与物流、配件备件、图文信息、认证资料、销售支持。
2. 不需要手动查询系统 ID。分类、商品计量单位、供应商、区域、客户类型、配置项、技术参数、可选配置等只填写名称，后续由系统工具解析真实 ID 并在提交草稿内部回填。配件/备件/易损件的「单位」是纯文本表单值，不解析 ID。
3. 表格中的 `配置项ID`、`参数ID`、`配置ID`、`选项ID` 仅供系统回填，普通用户保持空白。创建商品时不要填写媒体、认证、客户案例、配件、价格阶梯等行级 `id`。
4. 文件类字段填写相对路径，路径相对于本 Markdown 文件所在目录，例如 `./图片/主图.jpg`、`./认证/CE证书.pdf`。
5. 多个文件一行一个，或在表格中一行一个；不要把多个文件路径写在同一个单元格里。
6. 金额、重量、尺寸、数量、天数只写数字，不带单位。
7. 是否类字段只填写 `是`、`否` 或留空。
8. 日期统一填写 `YYYY-MM-DD`。
9. 状态字段支持 `上架`、`下架`、`作废`。不确定时留空；只有明确要求作废时才填写 `作废`。
10. 创建真实商品前，需要由维护人确认资料真实有效，并确认允许上传资料包文件。

> 条件性必填说明：
> - 产品类型为 **整机** 时，必须填写产品等级、参考成本价 人民币、利润率 %，并在“2.1 基础配置”填写至少一行整机配置；已填写的基础配置/技术参数必须补全配置值/参数值。
> - 产品类型为 **整机或配件** 时，必须填写包装长/宽/高 mm、包装费、包装重量 kg、净重 kg。
> - 适用范围为 **指定区域** 时，必须在“1.3 地域信息”填写至少一行区域明细。
> - 产品型号可留空；一旦填写，仅允许英文大小写字母、数字与空格，不能含首尾空格、中文或其他符号。

> 以下字段不作为硬阻塞必填项，按业务实际情况填写，留空不会阻塞创建：商品英文名称、Banner 图、包装方数。

### 0.2 内容丰满度建议

这些不是通用硬拦截，但会直接影响商品详情页是否空洞。资料充分时建议尽量填写：

| 模块 | 建议填写量 |
|---|---|
| 商品图片 | 主图 1 张；Banner 图 1 张；细节图、尺寸示意图、场景图、包装图、多角度实拍图、配件图按资料各填多行；自动整理时优先使用原始表格分类或直接父目录名，不得主观改写 |
| 商品视频 | 有实拍、作业、装柜、安装、包装、链界实测、三方实测资料时分别填写；若目录名与标准分类不完全一致且无明确映射，保留原目录名并在备注标记 |
| 图文详情卡片 | 至少 3 条，包含标题、副标题、正文、图片 |
| 核心优势 | 建议 3-5 条 |
| 应用场景 | 建议 3-5 条 |
| 常见问题与标准回答 | 建议 5 条以上 |
| 客户案例 | 有真实案例时建议 1 条以上，每条至少 1 张图片 |
| 认证资料 | 有证书、检测报告、说明书时按份填写 |
| 配件/备件/易损件 | 有清单时分别填写，不混在一个单元格 |
